== Errors found this session ==

1. FRONTEND — dashboard was not live (fixed)
   The five fleet dashboard hooks (useDevicesQuery, useAlertsQuery, useOverviewQuery,
   useHealthQuery, useRecoveryActionsQuery) had no refetchInterval, so the Fleet Monitor,
   alerts, and overview never auto-refreshed; only audit logs/incidents (20s) and the
   device-detail page (10-15s) polled. Multiple devices streaming telemetry looked frozen
   without a manual refresh.
   Fix: refetchInterval added — devices/alerts 15s, overview/health/recoveryActions 20s.

2. ANDROID — back-navigation regression + 7-item bottom nav (fixed, shipped as v2.0.1)
   Details in SentinelX_Andriod/Evidence/2026-07-13-uiux-audit-v2.0.1/01-audit-findings.md

3. DOCS — INSTALL_GUIDE.md still described v1.0.0 (fixed)
   APK path, Live Monitor modes (old 15/30/60s vs actual 60/30/10s), and the ingest-time
   timestamp limitation were all stale against v2.x. Guide rewritten; new
   LIVE_TELEMETRY_GUIDE.md added for the end-to-end multi-device flow.
